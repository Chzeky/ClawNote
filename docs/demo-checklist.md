# ClawNote 答辩演示与回归清单

## 演示目标

- 用 3 到 5 分钟展示 ClawNote 从知识采集、AI 整理、入库、检索、问答、图谱到推荐的完整闭环。
- 默认使用 PC 端 Web 页面作为主展示入口，飞书作为多 Agent 全链路补充证据。
- 网络不稳定时优先使用本地 SQLite 演示数据，避免把答辩节奏绑定到外部模型或网页抓取。

## 建议演示数据

## PC 端回归顺序

1. 打开 `http://localhost:5173`，确认工作台统计、最近入库、分类分布和高频主题正常显示。
2. 点击“新增知识”，分别验证粘贴文本、网页链接、上传文件三种入口的表单状态。
3. 用一段 200 字左右演示文本执行“AI 整理”，确认处理中状态、草稿预览和“确认入库”流程。
4. 打开新知识详情，验证标题、摘要、分类、标签、来源和正文展示。
5. 编辑新知识标题或标签，保存后确认列表与详情同步更新。
6. 搜索中文关键词和英文关键词各一次，截图保留搜索结果。
7. 在智能问答中提问，确认回答包含引用按钮，并能打开引用详情。
8. 切换到知识图谱，先展示分类气泡，再进入分类、知识、主题层级。
9. 切换关系网络图，确认知识节点和主题节点显示正常。
10. 切换相关推荐，选择不同知识，确认推荐理由和相似度显示。
11. 删除临时测试知识，确认二次确认弹窗和删除后列表刷新。
12. 停止后端或断网场景下，确认页面出现失败提示、超时提示或“重试”入口。

## 飞书链路验证

- 向飞书机器人发送一次完整知识整理请求。
- 确认 steward 调用 collector、organizer、graph、qa、recommender 的链路证据。
- 答辩中只展示关键截图，避免现场等待模型或跨 Agent 调度。
