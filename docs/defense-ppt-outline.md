# ClawNote 答辩 PPT 大纲

建议控制在 10 到 12 页，整体叙事为“痛点 → 目标 → 架构 → 核心功能 → 难点与测试 → 总结展望”。

## 第 1 页：封面

- ClawNote 个人智能知识管家
- 基于 OpenClaw 的多 Agent 知识管理系统
- 组名、成员、指导教师、日期

## 第 2 页：项目背景

- 个人知识散落在网页、笔记和文件中。
- 手工分类和整理成本高。
- 普通搜索不能理解知识语义。
- 内容之间缺少结构化关系。
- 大模型直接回答容易产生幻觉，需要可追溯来源。

## 第 3 页：项目目标

- 多源知识采集。
- AI 自动整理标题、摘要、分类和标签。
- 统一 SQLite 知识库存储。
- RAG 智能问答并展示引用来源。
- 知识图谱展示内容关系。
- 基于标签相似度推荐相关知识。
- 多 Agent 自动分工协作。

## 第 5 页：系统架构

建议画成：

```text
React 前端 / 飞书机器人
          ↓
     FastAPI / steward
          ↓
collector → organizer → graph → qa → recommender
          ↓
 SQLite + FTS5 知识库
          ↓
      DeepSeek API
```

讲解重点：

- Web 前端负责可视化、CRUD、问答和图谱展示。
- 飞书机器人作为多 Agent 调度入口。
- FastAPI 统一对外提供 REST API。
- SQLite + FTS5 提供本地知识库存储和检索。

## 第 6 页：Agent 设计

| Agent | 职责 |
|---|---|
| steward | 接收用户需求、拆分任务、调度子 Agent、汇总结果 |
| collector | 采集文本、网页、文件和 RSS 内容 |
| organizer | 生成标题、摘要、分类、标签并写入知识库 |
| graph | 提取实体关系并生成知识图谱数据 |
| qa | 检索知识库并执行 RAG 问答 |
| recommender | 根据标签和内容相似度推荐知识 |

讲解重点：

- steward 只做规划和调度。
- 各 Agent 职责单一，便于定位失败环节。
- Agent 之间使用结构化结果传递，不依赖口头描述成功。

## 第 7 页：核心数据流程

```text
采集 → 清洗 → AI 整理 → 草稿预览 → 确认入库
                         ↓
                 图谱 / 问答 / 推荐
```

安全与可靠性：

- 使用参数化 SQL 防止注入。
- URL 采集拒绝本机、内网和危险重定向。
- 输入长度限制和文件大小限制。
- 配置与密钥分离，不把真实 token 写入仓库。
- 前端提供加载、错误、超时和重试状态。

## 第 9 页：关键技术难点

| 难点 | 解决方案 |
|---|---|
| 多 Agent 调度容易虚构成功 | 要求真实工具结果，steward 只汇总实际返回 |
| SQLite FTS5 中文检索有限 | FTS5 优先，中文未命中时使用参数化 LIKE 兜底 |
| VPN、代理和外部 API 不稳定 | 本地知识库优先展示，前端增加超时与重试提示 |
| 网页正文容易混入导航和公式 SVG | 优先提取 main/article，忽略脚本、样式和 SVG 内部节点 |
