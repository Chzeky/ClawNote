# ClawNote 开发日志

## 2026-07-18：项目设计与 Agent 拆分

- 确定选题为“个人智能知识管家”。
- 将系统拆分为 steward、collector、organizer、graph、qa、recommender 六个 Agent。
- 决定由 steward 作为飞书唯一入口，其他 Agent 作为专业执行单元。
- 为避免职责交叉，明确每个 Agent 的输入、输出、异常处理和行为边界。

## 2026-07-18：飞书渠道与网络问题

- 问题：WSL 无法直连飞书和 DeepSeek，长连接和模型请求出现超时。
- 原因：WSL 直连不可用，Gateway 服务未正确使用代理；代理最初允许访问 loopback。
- 解决：为 OpenClaw 配置代理，并拒绝 `127.0.0.0/8`、`169.254.0.0/16`，代理验证通过。
- 结果：飞书机器人可以接收消息并路由到 `clawnote-steward`。

## 2026-07-18：多 Agent 调度问题

- 问题：steward 声称调用 organizer，但没有真实工具记录。
- 原因：缺少 `sessions_spawn`、`sessions_yield`、`subagents` 权限，且 Agent 出现过虚构调用。
- 解决：增加子 Agent 白名单和工具真实性规则；没有工具结果时禁止声称调用成功。
- 结果：steward 可以通过真实 Agent 调用获得 organizer 结果。

## 2026-07-19：Skill 结构调整

- 问题：原有 Skill 是平铺 Markdown，OpenClaw 无法自动识别。
- 决策：将 9 个细分说明合并为 5 个综合 Skill。
- 结构：每个业务 Agent workspace 下使用 `skills/<name>/SKILL.md`。
- 验证：`openclaw skills info <name> --agent <id>` 返回 `Ready`。

## 2026-07-19：SQLite 知识库

- 选择 SQLite，原因是零服务部署、便于演示、支持事务与 FTS5。
- 使用 `knowledge_items` 保存正文、摘要、分类、标签和来源。
- 使用 `knowledge_fts` 提供全文检索，并通过触发器同步索引。
- 使用 Python `sqlite3` 的 `?` 占位符实现参数化查询，防止 SQL 注入。

## 2026-07-19：中文检索兼容

- 问题：SQLite FTS5 的 `unicode61` 对中文子串检索支持有限。
- 解决：保留 FTS5 为首选检索；零结果时使用带转义的参数化 `LIKE` 查询标题、正文、摘要和标签。
- 结果：中文关键词可以命中，返回值中的 `search_mode` 会注明 `fts5` 或 `like_fallback`。

## 2026-07-19：Web API CRUD 闭环

- 新增：知识详情、部分更新和删除接口，前端无需执行终端命令即可完成完整 CRUD。
- 安全：动态更新列只来自 Pydantic 固定模型，字段值与知识 ID 继续使用 SQLite 参数化绑定。

## 2026-07-19：Web 智能导入

- 需求：减少手动填写标题、分类、摘要和标签的操作，允许用户直接粘贴整段知识原文。
- 流程：前端提交原文到分析接口，organizer 以 `draft` 模式返回结构化草稿；用户预览并修改后，才调用原有新增接口入库。
- 安全：正文按不可信 JSON 数据传递，提示 organizer 不执行正文内指令；后端使用无 shell 的参数数组启动 OpenClaw，并用 Pydantic 严格校验输出字段和长度。
- 隔离：分析接口不调用数据库写入工具。真实联调前后知识条目数均为 6，证明生成草稿不会自动入库。
- 验证：桌面端与 390×844 移动端布局通过；React lint、Vite 构建、21 项 Python 测试和 19 项 Jest 测试全部通过。

## 2026-07-19：多源 Web 导入

- 新增：智能导入扩展为粘贴文本、网页链接、上传文件三种入口，统一进入 organizer 草稿预览，再由用户确认入库。

## 2026-07-19：AI 整理性能优化

- 现象：强化学习教程页面抓取正文约 4870 字，抓取成功但 organizer 首次整理约需 11.2 秒；开发期间 Vite 热更新还会清空尚未提交的表单，看起来像页面自动变空。
- 对照：复用常驻 Gateway 的请求超过 100 秒后超时，因此继续使用稳定的 OpenClaw `--local` 调用。
- 优化：关闭额外 thinking，约束摘要为单段精简输出，并加入 64 条基于正文和标题提示哈希的内存 LRU 草稿缓存；前端分别显示“正在抓取网页”和“AI 正在整理”。
- 结果：目标页面首次整理约 6.6 秒，相同内容再次整理约 0.002 秒；缓存随后端重启清空，不写入数据库。
- 稳定性：新增表单状态写入 `sessionStorage`，开发热更新或意外刷新后恢复导入方式、网址、原文和 AI 草稿；关闭抽屉或成功入库后自动清除。

## 2026-07-19：Web RAG 智能问答

- 架构：后端先以标题、标签、摘要和正文的加权规则检索 SQLite，再将有限证据交给 `clawnote-qa`，避免 Agent 自行重复检索或调用无关工具。
- 约束：问题和证据均作为不可信 JSON 数据传入；qa Agent 只能基于证据回答并使用 `[知识 #编号：标题]` 引用，输出由 Pydantic 严格校验。
- 防幻觉：无相关证据时直接返回无法回答，不调用模型；引用编号、标题和来源 URL 由后端生成而非相信模型输出。
- 前端：新增“智能问答”视图、对话记录、置信度、引用按钮和处理中状态；点击引用可直接打开对应知识详情。
- 实测：问题“强化学习和有监督学习的主要区别是什么？”在约 7.4 秒内返回，仅引用知识 #9；桌面端和 390×844 移动端无报错或横向溢出。
