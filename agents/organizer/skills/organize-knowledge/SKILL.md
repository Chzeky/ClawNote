# 知识整理

## 输入

一条未经整理的知识内容。

## 工作流程

1. 提取核心观点。
2. 生成简洁摘要。
3. 判断知识分类。
4. 生成三至五个标签。
5. 将整理后的内容写入个人知识库。
6. 返回知识条目的唯一编号和存储状态。

## 输出

返回知识条目编号、标题、摘要、分类、标签、整理后的正文、来源和存储状态。

## 规则

摘要必须忠于原文，不得补充原文没有的事实。

## 存储规则

知识条目必须保存标题、正文、摘要、分类、标签、来源和创建时间。

写入成功后返回 `knowledge_id` 和 `stored: true`；写入失败时必须返回失败原因，不得声称已经保存。

## 知识库存储

完成摘要、分类和标签生成后，必须调用：

`python3 ../../scripts/knowledge_db.py add`

需要传入：

- `--title`
- `--content`
- `--summary`
- `--category`
- 一个或多个 `--tag`
- `--source`

不得直接拼接或执行 SQL。

只有工具返回 `success: true`、`stored: true` 和 `knowledge_id` 后，才可以告诉用户知识已保存。

如果写入失败，必须返回错误原因，不得声称保存成功。
